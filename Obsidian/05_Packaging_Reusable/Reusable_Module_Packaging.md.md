## ‚ö° UV Packaging Requirements
- [ ] **Package Name:** The package must be called `mazegen-*` and the file must be located at the root of your git repository.
- [ ] **Allowed Formats:** Both `.tar.gz` and `.whl` extensions are allowed, as generated by the standard build of a Python package  (e.g., `mazegen-1.0.0-py3-none-any.whl` ).
- [ ] **Build Source (uv):** We must provide in our Git repository all needed elements to build the package. This means setting up a proper `pyproject.toml` that `uv` can use to build the module.
- [ ] **Evaluation Ready:** During the evaluation, we will be asked to install the needed tools in a virtualenv or equivalent, and build the package again from our sources. We must be ready to demonstrate this using `uv` commands (like `uv venv`, `uv sync`, and `uv build`).
